## 2. Claude Codeによる検証手法の分類

### 手法A: デスクリサーチ（Web検索+情報分析）
Claude Codeがウェブ検索・公開データ分析で事実を確認する。最もコストが低く、最初に実施すべき。

### 手法B: ペルソナシミュレーション（AI人格による反応テスト）
Claude Codeが複数のペルソナ（EC事業者、歯科院長、配偶者等）を演じ、サービス提案・価格・メッセージングへの反応をシミュレーションする。リアル商談の予行演習。

### 手法C: 構造分析（ロジック検証+計算検証）
Claude Codeが計画内の数値整合性、工数見積もりの妥当性、財務モデルの感度分析を行う。

### 手法D: 敵対的検証（プレモーテム+レッドチーム）
Claude Codeが意図的に「この事業が失敗する理由」を探し、計画の盲点を発見する。

### 手法E: 類似事例分析（ベンチマーク調査）
Claude Codeが類似の1人AI事業・SMB向けコンサルの実例を調査し、成功/失敗パターンを抽出する。

## 4. 検証の実施計画（5ラウンド構成）

### ラウンド1: 事業の土台を検証する（デスクリサーチ集中）

**目的**: 事業計画の前提となる「市場は存在するか」「競合空白は本当か」「営業チャネルは機能するか」を事実ベースで確認する。ここが崩れれば計画全体の見直しが必要。

**検証項目と具体的方法**:

#### R1-1. 競合空白地帯の検証（IH3）— 最重要

| 項目 | 内容 |
|------|------|
| **検証する仮説** | ¥5-15万/月 × 業種特化 × AI伴走型のゾーンに競合がほぼいない |
| **手法** | Web検索で以下を網羅的に調査 |
| **調査対象** | ①「EC AI コンサル 月額」②「クリニック AI導入 支援」③「SMB AI 伴走」④「Shopify AI 最適化 サービス」⑤「歯科 DX コンサル」等のキーワードで競合を探索。Shopify App Store、ココナラ、ランサーズ、ビザスク、ITreview等のプラットフォームも確認 |
| **確認ポイント** | (a)同価格帯の競合は何社あるか (b)彼らの提供価値・差別化ポイントは何か (c)2025-2026年に新規参入した競合はいるか (d)大手コンサル（Funai等）が価格帯を下げてきていないか |
| **Go/見直し基準** | Go: 直接競合3社以下 → 空白地帯確認。見直し: 5社以上 → 差別化戦略の再設計が必要 |

#### R1-2. Shopify Partner制度の実態検証（IH9）

| 項目 | 内容 |
|------|------|
| **検証する仮説** | Shopify Partner経由で月1-3社リードを獲得できる |
| **手法** | Shopify Partner Programの公式情報+日本のパートナー事例を調査 |
| **確認ポイント** | (a)登録要件と審査期間の実態 (b)パートナーに紹介が来る仕組みの詳細 (c)日本のShopify Partnerの体験談・成功事例 (d)パートナー飽和度（日本に何社いるか） (e)実際にリードが来るまでの期間 |
| **Go/見直し基準** | Go: 制度が機能し、月1社以上のリード実績あり。見直し: リード獲得困難 → チャネル優先順位を変更 |

#### R1-3. IT導入補助金2026の確認（IH8）

| 項目 | 内容 |
|------|------|
| **検証する仮説** | IT導入補助金が営業レバーとして使える |
| **手法** | IT導入補助金2026の公式情報+最新ニュースを調査 |
| **確認ポイント** | (a)2026年度の制度継続は確定しているか (b)補助率・上限額・対象経費 (c)IT導入支援事業者の登録要件と審査期間 (d)AI関連ツールは補助対象か (e)申請〜採択〜入金のタイムライン (f)「月額課金型」サービスは対象になるか |
| **Go/見直し基準** | Go: 制度継続+AI対象+月額課金OK → 営業メッセージに組み込み。見直し: 制度廃止 or AI除外 → 補助金なしでの営業戦略に修正 |

#### R1-4. Shopify日本市場の規模検証（IH1）

| 項目 | 内容 |
|------|------|
| **検証する仮説** | 日本38,000+店舗、Tier A該当5,700-7,600店 |
| **手法** | Shopify Japan公式発表・調査レポート・業界記事を検索 |

| **Go/見直し基準** | Go: TAMが5,000店以上。見直し: 3,000店未満 → ターゲット定義を拡大 |

### ラウンド2: 顧客の反応をシミュレーションする（ペルソナ検証）

**目的**: 実際の商談に入る前に、「この価格でこのメッセージを言ったら、このペルソナはどう反応するか」をAIでシミュレーションし、メッセージングと価格設計を磨く。

**ペルソナシミュレーションの設計方針**:
- 各ペルソナは独立したプロンプトで生成し、事業計画の内容を知らない状態からスタートする
- 楽観的なペルソナだけでなく、懐疑的・否定的なペルソナも含める
- 各ペルソナに同じ営業ピッチを行い、反応のパターンを分析する

#### R2-1. EC事業者ペルソナパネル（H1, H2, IH4検証）

**作成するペルソナ（5名）**:

| ペルソナ | プロファイル | 性格特性 | 検証する仮説 |
|---------|-----------|---------|-------------|
| EC-A: 鈴木美咲（原型） | Shopify・月商800万・アパレル・36歳 | 好奇心旺盛だが慎重 | H1, H2のベースライン |
| EC-B: 田中大輔 | Shopify・月商350万・食品・42歳 | ITに疎い。コスト意識が強い | H1の下限テスト（¥5万でも高いか） |
| EC-C: 佐藤ゆり | BASE・月商1,500万・雑貨・33歳 | データ好き。自分でChatGPT活用中 | H2検証（PGより広告最適化が優先？） |
| EC-D: 中村健二 | Shopify・月商500万・家電・50歳 | 保守的。過去にコンサルで失敗経験 | H1の懐疑テスト（コンサル不信） |

**シミュレーション手順**:
1. 各ペルソナを詳細設定（1日のスケジュール、痛み、予算感、AI認知度）
2. 営業メッセージ3パターンを提示し、各ペルソナの反応を取得:
   - パターンA: 「ROAS+30%の実績で広告費を最適化」（実績訴求）
   - パターンB: 「商品ページ作成を1/6に短縮」（ジョブ訴求）
   - パターンC: 「パート1人分の月額で、AI2人分の仕事」（コスト比較訴求）
3. 価格提示テスト: ¥5万 → ¥10万 → ¥18万 の順に提示し、各価格での反応を取得
4. 反対意見・懸念事項を洗い出し、対策を設計

**評価指標**:
- 5名中何名が「検討したい」と反応するか（H1: 3名以上でGo）
- 最も刺さったメッセージパターンはどれか
- 最大の懸念事項は何か（未知のオブジェクションの発見）
- H2: 5名中の最大ジョブは本当に「商品PG高速化」か

**目的**: 計画内の数値前提・工数見積もり・財務モデルにおかしな点がないかを検証する。

#### R3-1. 週次実行計画の工数リアリティチェック（IH19, H7）

## 7. 検証の限界（Claude Codeでは分からないこと）

以下は必ずリアル検証（実際の人間との接触）が必要:

| 仮説 | なぜClaude Codeでは不十分か | 最小限のリアル検証方法 |
|------|-------------------------|-------------------|
| H1（WTP）の最終確認 | AIペルソナは「お金を払う」行動を再現できない | MVP診断5社（現計画通り） |
| H3（紹介チェーン） | 紹介行動は社会関係に依存 | 1社目の成功後に観察 |
| H7（品質維持）の実証 | 実稼働なしには分からない | パイロット3ヶ月 |
| 実際の商談の空気感 | テキストでは非言語情報が欠落 | 初回Zoom商談2-3件 |
| 配偶者の実際の反応 | ペルソナは推測にすぎない | 商談時に同席依頼 |

**したがって、Claude Code検証で「仮説の精度を上げる」ことが目的であり、「仮説を確定する」ことはできない。リアル検証の代替ではなく、リアル検証の前に勝率を上げるための準備作業と位置付ける。**
