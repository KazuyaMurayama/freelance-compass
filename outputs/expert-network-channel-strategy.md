## 1. 概要と位置づけ

### 1.1 本ドキュメントの目的

事業計画 v2.1.1 において Expert Network（ビザスク/GLG/AlphaSights 等のスポットコンサル系プラットフォーム）は、L1/L2/L3 の主力サービスを補完するキャッシュフロー早期化・リード補完チャネルとして位置づけられている。本ドキュメントは、その登録から定常化・拡張までの実行計画を具体化し、月稼働32時間制約下でROIを最大化する運用ルールを定める。

なお、v2.1.1 における L1 獲得主動力は「ECzine 寄稿 / ビザスク / 知人紹介」の並列であり、Expert Network はそのうちの1経路に過ぎない点を明示する。本チャネルを過大評価せず、Kill 基準で機械的に運用する。

### 1.2 事業計画 v2.1.1 における位置づけ

| 項目 | v2.1.1 記載内容 | 本戦略での具体化 |
|---|---|---|
| 登録時期 | M3 前倒し | M1中の登録申請、M2末までに公開・初件受注 |
| 月工数 | 平均 4h/月 | Phase 0（M1-M3）は立ち上げ期につき 6h/月（うち2hはL1営業時間からの一時振替）、Phase 1以降は 4h/月へ収束 |
| 売上目標 | M3: ¥3万、M6以降: ¥10-15万/月 | M3: 1件/月、M6: 3件/月、M12: 3-5件/月 |
| 二次機能 | L1/L2リード発掘経路（補完） | セッション→L1興味表明率 ≥10%、L1受注 累計M12までに3件 |
| 拡張 | M12以降にGLG/AlphaSights追加 | M9から英語プロフィール準備、ビザスク実績15-20件をベースにM12申請 |

### 1.3 二重機能の整理

Expert Network は単発収益源とリード補完の2機能を担う：

1. **月次CF早期化機能**
   - 受注から入金まで概ね30-45日。L1（3ヶ月PoC）の長い回収サイクルと比べ月次CFを早期に底上げ。
   - 事業初期（M1-M6）の「収益の谷」を埋める補完チャネル。固定費（社保・ツール代）相当の一部 ¥3-10万/月を確保。

2. **リード補完機能（L1/L2の入口の一つ）**
   - EC事業者のCxO/事業責任者から直接相談を受ける場合、相談者そのものが理想的なICP（Ideal Customer Profile）に近い。
   - 60分セッションで信頼を構築し、相手からの後日相談を受ける形でL1/L2へ自然遷移。
   - 営業コストゼロでICPと1on1できる稀少チャネル。ただし全相談者がICP合致するわけではない点を前提に置く。

### 1.4 成功の定義（North Star）

- **収益面**: M6時点で Expert Network 売上 ¥10万/月以上を3ヶ月連続達成（月3件×平均手取り¥3.5万）
- **リード面**: M12までに セッション経由のL1受注 累計1件以上（M6時点ではL1興味表明1件以上）
- **学習面**: 各セッションで「ICPの真のWTP」「競合(KARTE/b→dash等)の使用実態」「未解決の業務課題」を3点以上ヒアリング完了

### 2.2 優先順位の根拠

**第1優先（M1）: ビザスク本体のみ単独登録**
- 国内案件数No.1、EC領域・AI/DS領域とも案件流入が安定。
- 「EC事業者×AI活用」というニッチが日本市場にフィット（GLGは大手向け過ぎる）。
- 即日プロフィール公開可能、初件まで2-4週間が業界標準。
- ビザスクライトはM3以降に「純収益用途」として追加検討。M1同時登録は工数分散になるため見送り。

**ビザスクライトを後ろ倒しする理由**
- ライト単価¥1-3万はL1のICP（CxO層）と客層が乖離。リード機能を期待できない。
- 本体での運用ノウハウを確立してから追加した方が学習効率が高い。

**第2優先（M12）: GLG**
- 単価が3倍。月1件でも¥15万。
- 英語プロフィール整備（M9開始）と、ビザスクでの実績ログ（M12時点で累計15-20件想定）を申請時の信頼担保に使う。
- 実績件数が不足の場合は GLG 申請を M15 以降に再スケジューリング。
- PE/HF系クライアントは「日本のEC市場×AI浸透度」をテーマに調査することが多く、Amazon Ads経験 + LightGBM/SHAP の組み合わせは希少。

**第3優先（M13以降）: AlphaSights / Guidepoint**
- GLG稼働が安定したらクロスリスト。同じ準備で増分収益。
- Guidepoint は日本オフィスがあり英語負荷が GLG/AlphaSights より低い。

**不採用: HireAI / 一般クラウドソーシング**
- 単価¥2-5万でL1（¥6万/月×3=¥18万）とカニバライズ。
- 客層が「業務を切り出して安く頼みたい」層で、L1のICP（経営判断にAIを使いたいCxO）と乖離。

### 4.5 L1/L2への継続支援案内（規約準拠）

**前提:** ビザスク規約上、プラットフォーム外契約への直接誘導は禁止行為。「情報提供にとどめる」「具体額や契約条件は伝えない」「相手からの問い合わせを受ける形にする」「再相談もビザスク経由を歓迎する」が鉄則。

**自然導線スクリプト例（規約準拠版）:**

> 「本日お話しした需要予測の検証について、もし社内で実装まで進めるご検討があれば、私自身も個別の支援プログラムをご用意しています。具体的なご相談があれば、ビザスク経由で改めてお声がけいただければ承ります。」

**やってはいけないこと（NG例）:**
- 具体的な月額単価（¥6万、¥17万、¥25万）をセッション中に口頭で伝える
- IT導入補助金の活用額を金額付きで提示する
- 「ビザスクを通さず別途連絡してほしい」と示唆する
- 名刺やメールアドレスの直接交換を持ちかける

**ポイント:**
- 売り込まない。「選択肢の一つ」として情報提供にとどめる
- 具体的な金額・条件は、相手から問い合わせを受けた後に、ビザスク経由のメッセージで提示する
- 継続支援もビザスク経由の案件として受ける前提で設計（プラットフォーム手数料を負担しても継続関係は歓迎）

## 6. 仮説検証項目（hypothesis-verification-plan v3 整合）

### 6.1 IH10: ビザスクで即時リード補完が可能

**仮説:** ビザスク登録後6ヶ月以内に、L1見込み顧客（=セッション後にL1案内に対し「詳しく話を聞きたい」と返信した相談者）を1名以上獲得できる。

**検証方法:**
1. M1-M6 で実施した全セッションのフォローアップメッセージ送付状況をログ化
2. 返信内容を「L1興味あり / L1興味なし / 無反応」の3分類
3. 「L1興味あり」=1件以上なら IH10 検証OK

**KPI（統一基準）:**
- セッション → フォローメッセージ返信率: ≥40%
- フォローメッセージ返信 → L1興味表明率: ≥25%
- セッション → L1興味表明率（総合）: ≥10%
- L1興味表明 → L1受注率: ≥30%（=セッション→L1受注率 ≥3%）

**判断基準:**
- M3末: 興味表明1件以上 → Phase 1で強化 / 0件 → プロフィール抜本改訂
- M6末: 興味表明3件以上＋L1受注1件以上 → リード機能として継続 / 未達 → 純収益チャネルへ役割再定義
- M12末: L1受注累計3件未達 → リード機能期待を撤回、収益チャネルに純化

## 8. リスクと対策

### 8.1 プラットフォームリスク

| リスク | 影響 | 対策 |
|---|---|---|
| ビザスク規約違反による垢BAN（プラットフォーム外誘導と判定） | Expert Network チャネル消失 | 転換トークは「情報提供にとどめる」「具体額を口頭で伝えない」「主導権を相手に渡す」「ビザスク経由の継続発注を歓迎」を徹底。四半期ごとに規約変更を確認 |
| ビザスクの手数料率引き上げ | 手取り低下 | 表示単価の段階的引き上げで吸収、GLG/AlphaSightsへのポートフォリオ分散 |
| プラットフォーム自体の方針変更（EC領域案件減少） | 案件供給枯渇 | 複数プラットフォーム並行（Visasq/Lite/Mosaic/NewsPicks）でリスク分散 |
| アカウント審査落ち（GLG/AlphaSights） | M12計画遅延 | M9から英語プロフィール準備、ビザスク実績15件以上をエビデンスに使用、ダメなら Guidepoint に切替 |

### 8.2 時間単価悪化リスク

| リスク | 影響 | 対策 |
|---|---|---|
| 事前資料準備の肥大化 | 4h制約超過 | 「事前資料30分超は別見積」を最初に握る、雛形ライブラリ化で準備時間圧縮 |
| 1案件に複数回ミーティング要求 | 工数オーバー | 60分1回が標準、2回目以降は別案件扱い |
| 単価¥3万のライト案件ばかり | 時間単価悪化 | 月間受注の50%以上は本体（¥5万+）に維持、ライトは月1件まで |
| 「無料の続き」を求められる | 規約抵触+工数増 | フォローメッセージ1通で完結、それ以上はビザスク経由再依頼を案内 |

### 8.3 競合差別化

**競合エキスパートの典型タイプと本サービスのポジション:**

| 競合タイプ | 強み | 弱み | 本サービスの差別化 |
|---|---|---|---|
| 元事業会社マーケ責任者 | 現場知見、業界人脈 | データ/MLの実装力不足 | LightGBM/SHAP実装まで踏み込める |
| データサイエンティスト（受託系） | ML技術力 | EC業務知識不足 | Amazon Ads実務経験でドメイン理解 |
| ECコンサル（戦略系） | 経営目線 | 技術実装と乖離した提言になりがち | Streamlit即試作で「動くもの」を示せる |
| 元Amazon社員（営業/事業開発系） | Amazon内部知見 | データ分析側ではない | DS側スタンスで参入、ニッチ独占 |

**差別化メッセージ:**
「Amazon Ads実務×データサイエンス×Streamlit即試作」の三点交差は国内ビザスクエキスパート群でも稀少。プロフィール・実績記載でこの三点を一貫して打ち出す。

### 8.4 機会損失リスク（Expert Networkに時間を使うことのコスト）

- Phase 0 で月6h投下 = 月稼働32hの18.8%
- Phase 1以降で月4h投下 = 12.5%
- もしL1顧客獲得につながらない場合、同じ時間を「直接営業（SNS/紹介）」に振った方がROIが高い可能性
- **対策**: M6末のKill基準で機械的に判断、感情で継続しない
