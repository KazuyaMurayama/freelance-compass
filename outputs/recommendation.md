### 層0「休止プロトコル」（R2対策）

```

短期離脱（1-2週間）:
  → アプリは自動稼働。障害アラート→翌営業日対応をSLAに明記
  → 層1クライアントに事前通知「今週はチャット対応のみ」
  → 層2のZoomは録画配信に切替

長期離脱（1ヶ月以上）:
  → 層1は一時休止（月額も停止→信頼維持）
  → 層0は自動継続（課金も継続）
  → 層2はゲスト講師（士業パートナー等）に1回だけ依頼
```

### 「クローン戦略」: 知識の外部化（R2/R4対策）

自分の判断パターンをClaude Codeが再現可能な形で構造化する。

```
knowledge/
├── industries/
│   ├── legal.md          # 士業クライアントの典型課題・対応パターン
│   ├── ec.md             # EC事業者の典型課題・対応パターン
│   └── clinic.md         # クリニックの典型課題・対応パターン
├── templates/
│   ├── onboarding.md     # オンボーディングチェックリスト
│   ├── monthly_review.md # 月次レビューテンプレート
│   └── diagnosis.md      # AI診断レポートテンプレート
├── pricing/
│   └── negotiation.md    # 価格交渉パターン・回答例
└── CLAUDE.md             # Claude Codeへの指示書（上記を参照して判断）
```

**効果**: Claude Codeに `「EC事業者A社の月次レポートを作成して」` と指示するだけで、knowledge/ec.md + 過去の実績データを参照し、80%完成のレポートが30秒で生成される。人間はレビュー5分のみ。
