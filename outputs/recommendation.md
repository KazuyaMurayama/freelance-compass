## 3. v7.0 技術アーキテクチャ（★v6.0からの進化）

### 3-1. 「共通基盤+プラグイン」設計（R14対策の核心）

v6.0は個別アプリを1本ずつ構築する設計だった。v7.0では**共通フレームワーク1本+業種プラグイン**で構成し、保守コストを根本的に削減する。

```
ai-partner-platform/          # 共通基盤（1リポジトリ）
├── core/
│   ├── auth.py               # 認証・課金管理
│   ├── llm_router.py         # LiteLLM: Claude/GPT/Gemini自動切替（★R13対策）
│   ├── usage_tracker.py      # API使用量トラッキング+コスト最適化
│   ├── error_handler.py      # Sentry連携+Slack通知（★R14対策）
│   ├── tutorial.py           # アプリ内ウォークスルー（★R15対策）
│   └── terms_of_service.py   # 利用規約表示+同意管理（★R19対策）
├── plugins/
│   ├── legal_screening/      # 士業: 相談スクリーニング
│   ├── ec_copywriter/        # EC: 商品説明文生成
│   ├── clinic_reception/     # クリニック: 問い合わせ応答
│   └── template/             # 新規業種テンプレート（10分でプラグイン追加）
├── deploy/
│   ├── Dockerfile            # Docker化（★任意PaaS/VPSに移行可能）
│   └── railway.toml          # Railway設定
└── knowledge/                # クローン戦略DB
    ├── industries/
    └── templates/
```

**v6.0 → v7.0の変化**:
