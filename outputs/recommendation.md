### 3-2. 初期2業種の選定理由

**士業事務所を#1にした理由**:
- 競合スコア2（最低）= ブルーオーシャン
- 市場母数13万件超（税理士38k+社労士44k+行政書士50k）
- v7プラグインで「税理士」「社労士」「行政書士」を各0.5hで追加 → 3業種分のプラグインが1.5hで完成
- STANDARDでの大手5社AI活用推進経験が「定型業務の自動化」に直結
- 補助金申請は顧客側の士業と連携でき、紹介ネットワーク構築にもなる
- 士業は他の士業を紹介する文化がある → 口コミ拡散が期待できる

**クリニック・歯科を#2にした理由**:
- 需要スコア5（最高タイ）
- エムスリー4年+IQVIA経験で医療業界の規制・用語・現場感を理解（最大の参入障壁）
- v7のLLMルーターで月5万円プランが採算に乗る（旧Difyでは不可能）
- LINE連携AI問診+予約自動化で「電話対応50%削減・no-show率25%改善」の訴求が可能
- 院長は「忙しくてAIを調べる時間もない」→ 伴走型が刺さる

### 3-3. プラグイン展開ロードマップ（★v8.0新規）

```
Phase 0-1（Month 1-6）: 士業 + クリニック
  └─ プラグイン: 税理士/社労士/行政書士/クリニック受付/歯科問診 = 計5本（2.5h）

Phase 2（Month 7-12）: 工務店 + EC + 美容サロン
  └─ プラグイン: 工務店集客/EC販促/美容予約 = 計3本（1.5h）
  └─ 美容はbeauty-research-agentsの資産活用

Phase 3（Month 13-18）: 不動産 + 飲食
  └─ プラグイン: 不動産査定/飲食集客 = 計2本（1h）
  └─ 不動産はMachineLearning_Appの価格予測モデル転用

Phase 4（Month 19-24）: 勝ち業種の深掘り + 新規業種探索
  └─ 勝った業種に高機能プラグインを追加
```

**合計プラグイン数**: Phase 4完了時点で10本（構築工数わずか5h）

## 9. 判断基準（v8.0: 業種特化ゲート追加）

### Gate 0: Phase 0完了時（1ヶ月目末）
- STANDARDから副業承認を書面で取得したか → No: 対象範囲を再調整
- **★v8.0**: 士業5社中3社以上、またはクリニック5社中3社以上が有料意向 → No: もう5社追加
- **★v8.0**: 補助金訴求が刺さっているか → No: 訴求メッセージを調整

### Gate 1: 6ヶ月目
- 共通基盤が完成しているか → No: Phase 2に進まない
- 層0ストック ≧ ¥10万/月 → Yes: 層2開始OK
- **★v8.0**: 士業 or クリニックのどちらが勝ち業種か判明 → No: Phase 1延長
- 層1リテイナー ≧ 1社 → No: 業種ピボット検討（3位の工務店に切替）

### Gate 2: 12ヶ月目（最重要）
- 月収¥50万以上 → Phase 3へ
- 月収¥20〜50万 → Phase 2延長+改善
- 月収¥20万未満 → 業種ピボット

### Gate 3: 18ヶ月目
- 4層すべてが稼働 → Phase 4へ
- **★v8.0**: プラグイン10本以上が稼働 → 横展開成功の証拠
- 通算時給 ≧ ¥50,000 → 現行戦略継続
- 通算時給 < ¥50,000 → 最低ALR層を廃止
