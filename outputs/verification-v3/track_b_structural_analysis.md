### ブレークイーブン分析

**最低限の月収支均衡条件**:
- 月次コスト¥90,000をカバーするためのリテイナー最低社数: **1社（¥180k - ¥90k = +¥90k）**
- M1〜M3まで売上¥0でも持ち出しは¥90k×3 = ¥270k（個人の生活費は別途必要）

**24ヶ月で黒字にならないシナリオの条件**:
- 24ヶ月で1社もリテイナー獲得できない場合: 累計損失 ¥90k×24 = **-¥2,160k**
- プラグイン/塾のみの場合（コンサルなし）: 最大¥25k×8本 + ¥40k×6名 = ¥440k/月 → コスト控除後¥350k/月。持続可能だが目標に届かない

**重要**: 標準シナリオが最もリスクとなるのは**M1-M6の初期フェーズ**。リテイナー0社の期間が長いほど、心理的消耗と機会コストが増大する。

### ボトルネック発見（3シナリオ）

#### シナリオ1: 月末レビュー集中
```
問題: 4社全員が「月末最終週」を希望した場合
  Week 4: 4社×1.5h = 6h が1週に集中
  残り業務（営業・BIP・層0・層2）が翌月に溢れる

影響: 月次稼働が15h→20h超になる確率 約30%
対策: 月初・月中・月末に1社ずつ分散スケジューリング（A2が自動リマインド）
```

#### シナリオ2: 同時緊急対応
```
問題: 2社が同日に「AIエラー」「数値急落」等の緊急連絡
  月1回程度の発生確率で、各1〜2h の対応が必要
  通常タスクが翌週に押し出される

影響: 月次稼働+2〜4h/月（不定期）
対策: SLA設定（緊急対応は翌営業日以内、非緊急は72h以内）
    + A2の異常アラートで事前に問題を検知
```

#### シナリオ3: 1週間の不在（旅行・病気）
```
問題: 1人事業のため、不在時に代替者ゼロ
  月次レビューの延期依頼 → 顧客の不満リスク
  AIエージェントが一定程度をカバーできるか不明

影響: 月次解約リスク（不在の質低下が解約トリガーに）
対策: 月次レビューを「非同期（SlackメッセージでOK）」のオプションを設ける
    + 年2週の不在バッファをスケジュールに予め確保
```
